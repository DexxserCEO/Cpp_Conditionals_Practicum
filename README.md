# Cpp_Conditionals_Practicum
Практична робота по умовним операторам C++
Для чого використовується умовний оператор у мові програмування C++?
 Умовний оператор використовується для виконання певних дій (команд) тільки при виконанні заданої умови. Він забезпечує розгалуження алгоритмів у програмах.


Який загальний синтаксис оператора if? Загальний синтаксис оператора if:

C++

if (умова) {
    оператор;
}
Якщо від умови залежить лише один оператор, операторні дужки {} можна не ставити.



Чим відрізняється конструкція if від if...else?

if: Виконує оператор тільки тоді, коли умова істинна (true); якщо хибна (false), оператор пропускається.

if...else: Якщо умова істинна, виконується перший блок (if); якщо умова хибна, обов'язково виконується блок else.

Як працює структура if...else if...else? 
Ця конструкція дозволяє перевіряти послідовність умов. Перевірка відбувається зверху вниз. Виконується лише той блок коду, умова якого першою виявилася істинною (true), після чого решта блоків (включно з else) пропускається. Якщо жодна умова (if чи else if) не істинна, виконується фінальний блок else.


Як можна замінити простий умовний оператор if...else тернарним оператором, наведіть приклад.
 Тернарний оператор ?: (бінарний оператор умови) дозволяє присвоїти змінній значення на основі умови.

Синтаксис тернарного оператора: вираз = (умова) ? значення1 : значення2;.

Приклад заміни:

C++

// if-else
if (a > b) {
    max_val = a;
} else {
    max_val = b;
}

// Тернарний оператор
max_val = (a > b) ? a : b; // Якщо a > b істинна, max_val = a, інакше max_val = b
Які типи виразів можуть бути використані як умова в операторі if?
 У якості умови може бути використаний будь-який вираз, що повертає булеве значення (bool) або значення, яке може бути неявно перетворене на bool. У C++:

true (будь-яке ненульове значення) вважається істиною.

false (нуль) вважається хибою.

Що таке вкладені умовні оператори і коли доцільно їх використовувати?
 Вкладені умовні оператори — це конструкції, де один if, if...else або if...else if...else знаходиться всередині іншого блоку if або else.

Доцільність: Вони використовуються, коли потрібно перевірити другу умову, яка має сенс тільки тоді, коли перша умова вже була виконана. Наприклад, щоб перевірити, чи точка лежить у певній чверті координатної площини, спочатку перевіряють знак X, а потім, усередині цього блоку, перевіряють знак Y.

Яким чином можна забезпечити коректне виконання кількох операторів усередині блоку if? 
Для забезпечення коректного виконання кількох операторів усередині блоку if (або else if, else), необхідно використовувати операторні дужки (фігурні дужки) {}. Ці дужки створюють блок коду, який виконується як єдиний оператор.



Які можливі помилки можуть виникати під час використання умовного оператора?
 Типові помилки:

Випадкове використання оператора присвоєння (=) замість оператора порівняння (==) в умові.

Забування фігурних дужок {} при використанні кількох операторів усередині блоку if (призводить до того, що виконується лише перший рядок).

Логічні помилки в умовах, особливо при використанні логічних операторів (&&, ||) або при роботі з діапазонами.

"Висячий else" (Dangling else): Коли else помилково асоціюється з іншим if, ніж передбачалося (у C++ else завжди належить найближчому попередньому if без else).

Яку роль відіграють логічні оператори (&&, ||, !) в умовних виразах?
 Логічні оператори використовуються для комбінування або інвертування булевих умов:

&& (AND): Вираз істинний, тільки якщо обидві умови істинні.

|| (OR): Вираз істинний, якщо хоча б одна умова істинна.

! (NOT): Інвертує логічне значення умови (істина стає хибою, хиба — істиною).
